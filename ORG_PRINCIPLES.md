# Info-Tech-org 组织原则

我们是谁、信什么、怎么一起做事。  
**1NF0TECH（印否科技）** · [GitHub 组织](https://github.com/Info-Tech-org) · Backed by 深圳科创学院。

---

## 使命

**在伤害成型之前，把对话接住。**

我们做的是**家庭对话的语义安全**：不评判谁对谁错，只帮忙看见——语气什么时候过了线、哪些话会让孩子沉默、冲突之后怎么一起复盘。技术是手段，**人**才是目的：少一句后悔，多一次觉察。

---

## 愿景

- **眼下**：让有孩子的家庭真的用上——情绪上来时有提醒，吵完架后有可操作的复盘与建议，而不是「又这样了，不知道怎么办」。
- **长远**：让「孩子的记忆里不再有创伤」变成可衡量、可改进的事；和更多人一起，把家庭对话安全与心理健康写进行业里。

---

## 价值观

| 原则 | 我们怎么理解 |
|------|--------------|
| **人先于系统** | 产品与代码都为人服务：隐私与心理安全第一；声音与敏感数据不出家门，推理在边缘完成。 |
| **赋能，不裁判** | 帮家长看见自己、调整自己，不替代沟通、不判对错、不监控内容。我们信「觉察」比「纠正」更有用。 |
| **透明、可解释** | 识别逻辑、干预方式、数据流，对用户可说清；技术决策在文档里可追溯。不黑盒。 |
| **开放、可接** | 核心能力通过 API 与协议开放；欢迎在合规前提下接入、二次开发、一起折腾。 |
| **长期主义** | 扎在亲子对话安全这一件事上，持续磨算法、体验和生态，不追风口。 |

---

## 技术原则（极客那一面）

- **边缘优先**：语义识别与干预尽量在设备端跑，延迟低、隐私稳。云端能少则少。
- **管线可插拔**：关键词、语义向量、LLM 各是一层，可换可调。我们 ship 的是 pipeline，不是黑魔法。
- **多端一致**：Web、扩展、硬件（ESP32）、可穿戴——入口可以多，能力与策略要对齐，文档与协议先行。
- **文档即契约**：对外接口、数据格式、行为约定，用文档和版本化协议说清楚，方便协作和接入。

## 协作原则

- **尊重与包容**：遵守 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)；不同背景和观点都欢迎，讨论对事不对人。
- **贡献即认可**：报 Bug、改文档、提 PR、给产品反馈，都算贡献；我们按 [CONTRIBUTING.md](CONTRIBUTING.md) 认真看、认真回。
- **安全私下说**：漏洞和敏感问题走 [SECURITY.md](SECURITY.md)，不在此公开。
- **主仓与生态**：核心在 [Dialog](https://github.com/Info-Tech-org/Dialog)；其他仓（插件、模板等）和主仓规范对齐、职责互补。
