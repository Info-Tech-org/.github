# Info-Tech-org 组织原则

本文档阐明 **1NF0TECH（印否科技）** 在 [GitHub 组织 Info-Tech-org](https://github.com/Info-Tech-org) 下的使命、愿景、价值观与协作原则。Backed by 深圳科创学院。

---

## 使命

**在伤害成型之前，让对话回到安全区。**

我们致力于为家庭提供**对话语义安全基础设施**：通过实时识别与温和干预，帮助家长觉察沟通中的风险信号，减少无意识的言语伤害，支持亲子关系修复与共同反思。

---

## 愿景

- **短期**：有孩子的家庭能够用上我们的系统，在情绪升级前获得提醒，在冲突后有可操作的复盘与建议。
- **长期**：推动家庭对话安全与心理健康相关的行业共识与标准，让「孩子的记忆里不再有创伤」成为可衡量的目标。

---

## 价值观

| 原则 | 说明 |
|------|------|
| **安全优先** | 产品与代码以用户隐私与心理安全为前提；音频与敏感数据不离开家庭网络，推理在边缘完成。 |
| **赋能而非裁判** | 系统帮助家长觉察与调整自身行为，不替代沟通、不评判对错、不监控内容。 |
| **透明与可解释** | 识别逻辑、干预方式、数据流向对用户可理解；技术决策在文档中可追溯。 |
| **开放与可扩展** | 核心能力通过 API 与开放协议对外；欢迎社区在合规前提下接入与二次开发。 |
| **长期主义** | 专注亲子对话安全领域，持续迭代算法、体验与生态，而非追逐短期热点。 |

---

## 技术原则

- **边缘优先**：语义识别与干预尽量在设备端完成，降低延迟、保护隐私。
- **可插拔管线**：关键词、语义向量、LLM 等检测环节模块化，便于替换与优化。
- **多端一致体验**：Web、扩展、硬件（如 ESP32）、可穿戴等入口，在能力与策略上保持一致性。
- **文档与协议先行**：对外接口、数据格式、行为约定以文档与版本化协议形式维护，便于协作与接入。

---

## 协作原则

- **尊重与包容**：遵守 [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)；尊重不同背景与观点，建设性讨论技术与产品。
- **贡献即认可**：Bug 报告、文档改进、代码提交、产品反馈均被视为贡献；我们按 [CONTRIBUTING.md](CONTRIBUTING.md) 响应与合并。
- **安全敏感**：安全相关问题通过 [SECURITY.md](SECURITY.md) 约定渠道私下报告，不在此公开披露未修复漏洞。
- **主仓与生态**：当前核心代码与产品以 [Dialog](https://github.com/Info-Tech-org/Dialog) 为主仓；组织内其他仓库（如插件、模板）与主仓在规范上对齐，在职责上互补。
